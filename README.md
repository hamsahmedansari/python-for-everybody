# Python for Everybody

My solutions and worked examples for the [Python for Everybody](https://www.py4e.com/)
course by Dr. Charles Severance. The scripts are numbered roughly in the order the
material is covered, from basic syntax through web scraping, APIs, and databases.

## Layout

Numbered scripts (`00.py` … `23_db_03.py`) each map to an exercise or a worked example.
Chapter/exercise numbers are in the comment at the top of most files.

| Script | Topic |
|--------|-------|
| `00.py` | First program |
| `01_converting_user_input.py` | User input, conversion |
| `02`–`05` (`*pay*`, `score_range`) | Conditionals, functions, pay calculator |
| `06_loop.py` | Loops |
| `07_string.py` | String slicing and methods |
| `08`–`09` (`*file*`) | Reading files |
| `10`–`11` (`*list*`) | Lists |
| `12_dict.py` | Dictionaries |
| `13_tuple.py` | Tuples |
| `13_socket.py` | Sockets (Using Python to Access Web Data) |
| `14_regex.py` | Regular expressions |
| `15_urllib.py` | Fetching URLs with urllib |
| `16`–`17` (`*beautifulSoup*`) | HTML parsing and links with BeautifulSoup |
| `18_xml.py` | Parsing XML |
| `19`–`20` (`*json*`) | Parsing JSON |
| `21_db.py` | Counting organizations (SQLite) |
| `22_db_02.py` | Musical track database (SQLite) |
| `23_db_03.py` | Roster / many-to-many (SQLite) |

## Directories

- `geo-data/` — Chapter 16 GEOData project: geocode place names, store in SQLite,
  and visualize on an OpenStreetMap map. See [geo-data/README.md](geo-data/README.md).
- `data/` — sample input files used by some exercises (e.g. regex sum inputs).

## Supporting files

- `mbox-short.txt`, `mbox.txt` — mailbox sample data used across several exercises
- `tracks.csv`, `roster_data.json` — inputs for the database exercises
- `*.sqlite` — databases generated by the `21`–`23` scripts and the geo-data project
- `opengeo.zip` — original download for the geo-data assignment

## Running

Scripts use the Python 3 standard library. A few need third-party packages:

```bash
pip install beautifulsoup4
```

Then run any script directly:

```bash
python 12_dict.py
```

Some scripts prompt for an input file name (e.g. `mbox-short.txt`) or a URL; others
read a fixed file in the repo. On Windows, prefer **PowerShell** and set
`$env:PYTHONIOENCODING="utf-8"` when a script prints non-ASCII text.
